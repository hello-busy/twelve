# Contributing to Twelve

Thank you for your interest in contributing to the Twelve project! This guide will help you get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/twelve.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Create a pull request

## Project Philosophy

This project follows a minimalist approach:
- **Simplicity first**: Prefer simple solutions over complex ones
- **Minimal dependencies**: Avoid unnecessary external dependencies
- **Clear purpose**: Each feature should have a clear, focused purpose
- **Maintainability**: Code should be easy to understand and maintain

## Code Style

- Write clean, readable code
- Use meaningful names for variables and functions
- Follow the conventions of the chosen programming language
- Add comments only when necessary to explain complex logic
- Keep functions small and focused

## Pull Request Guidelines

- Fill out the pull request template completely
- Include tests for new functionality
- Update documentation as needed
- Ensure all tests pass
- Keep pull requests focused and atomic
- Reference related issues using "Closes #issue-number"

## Issue Guidelines

- Search existing issues before creating a new one
- Use the appropriate issue template
- Provide clear, detailed descriptions
- Include steps to reproduce for bugs
- Explain the motivation for feature requests

## GitHub Copilot

This repository is configured with Copilot instructions to help provide better coding assistance. The instructions can be found in `.github/copilot-instructions.md`.

## Questions?

If you have questions or need help, please:
1. Check existing issues and documentation
2. Create an issue with your question
3. Be patient and respectful in all interactions

We appreciate your contributions to making Twelve better!
